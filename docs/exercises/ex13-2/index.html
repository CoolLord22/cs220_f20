Exercise 13-2

info

As you work, feel free to discuss with other students and ask questions of the Professor/CAs. Aim to complete as much as possible during our meeting time. Work at home to complete what you do not get through during class. You will not hand in your work.

Learning Objectives#

This exercise gives you practice with expression evaluation, stacks, and exceptions.

Part 1#

Pull the starter code for this exercise from the public repo by taking the following steps:

  1. Log into ugrad, then navigate into cs220-f20-public, type git status and confirm you have not modified any files or accidentally committed to the public repo. Ask for help if you have added or committed files there and need to undo.

  2. Type git pull to synchronize your local repo and working copy with the remote repo.

  3. Confirm that you can see the template files for today‚Äôs exercise by typing ls exercises/ex13-2 -- you should see files named stack.h, stack.cpp, bounded_stack.h, and bounded_stack.cpp inside.

  4. Copy the above files from the public class repository (in the /exercises/ex13-2/ directory) to your personal repo in a new directory named ex13-2 under ~/cs220/exercises/.

Part 2#

In this part, you will write a program to evaluate a postfix expression comprising of numeric values and basic arithmetic operators (*, /, +, -). We are used to read infix expressions where operators come in-between the operands (e.g., 2.22 * 3.98 / 3.47 + 1). However, we may write an expression in prefix (operator comes before the operands) or postfix (operator comes after the operands) styles too. The postfix equivalent of the infix expression 2.22 * 3.98 / 3.47 + 1 is 2.22 3.98 * 3.47 / 1 +. The nice thing about postfix expressions is that neither precedence rules nor parentheses are needed (think about why!).

A typical program to evaluate postfix expressions uses a stack. Here is high-level pseudocode:

  • Create an empty stack and keep scanning the expression (from left to right) as long as there are more tokens to read:
    • If the encountered token is an operand (a number), push it onto the stack.
    • If the encountered token is an operator O, pop the stack twice to get numeric values A and B respectively. Calculate B O A and push the result back to the stack.
  • When the expression is ended, there should be only one numeric value in the stack showing the final result of the expression.

Example: 4 5 6 * +

Given the completed stack files from Exercise 13-1, write a program in a new file named postfix_interpreter.cpp to accomplish the above. Your program should ask the user to enter a postfix expression and then print the result. Be sure to make good use of the exceptions that the stack might throw to let you know something has gone wrong!

Note 1: If we

  1. end up having more than one value left in the stack at the end of the expression,
  2. try to pop an empty stack (i.e., underflow_error), or
  3. encounter invalid values and/or types or operator(s), then it means that the entered expression is invalid. Your code should account for these scenarios and show the message ‚Äúinvalid expression!‚Äù.

Note 2: Account for the case where a division by zero happens and show the message ‚Äúdivision by zero occurred!‚Äù.

Part 3#

Modify the program above to repeatedly ask the user for a line containing a postfix expression and show the results, one at a time. The program should terminate when the end of input is reached. Note that in each iteration you need to start with a fresh (i.e., empty) stack.

Part 4#

Now, modify your program to work with a bounded stack. If, during the evaluation of an expression, we exceed the stack capacity, you need to print a message ‚Äútoo many operands on the stack!‚Äù. Make use of the overflow_error exception that the class already throws.
